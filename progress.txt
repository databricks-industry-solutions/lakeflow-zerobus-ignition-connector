[iteration 8] 2026-02-13 - AGL example scripts + demo app readiness assessment

## Analysis completed

### Key findings

1. **Example scripts in examples/agl_tomago_bess_site01/** are designed for:
   - Ignition Designer tag import (JSON files)
   - Gateway Timer Scripts (Python simulation code)
   - Tag paths like `[agl_bess]AGL/Australia/NSW/Tomago/Site01/BESS01/Telemetry/SoC_pct`

2. **Two different data flows exist:**
   - **PRD flow**: demo/simulator (Node.js) → Zerobus SDK → raw_tags (PRD schema)
   - **Ignition flow**: Ignition → Zerobus Connector (protobuf) → tag_events (protobuf schema) → silver views

3. **Schema compatibility:**
   - Zerobus Connector outputs `tag_path`, `numeric_value`, `tag_provider` to `agl_ignition.scada_data.tag_events`
   - PRD expects `asset_id`, `asset_type`, `tag_name`, `tag_value` in `raw_tags`
   - Site SQL (`pipelines/sites/agl_tomago/`) bridges this with `silver_signal_mapping` + `silver_events_normalized`

4. **The demo app (`demo/app/`) queries PRD-schema tables**, not the Ignition connector output directly. To use Ignition example data with the demo app, the silver views provide the transformation layer.

## Ignition Designer workflow summary

1. Create tag providers: agl_bess, agl_grid, agl_market, agl_cmms
2. Import JSON tag files into each provider
3. Create 4 Gateway Timer Scripts (copy Python code from timer_script_*.py files)
4. Configure Zerobus Connector module with connection details
5. Run SQL setup: 10_silver_scaffolding.sql → 11_seed_tomago_site01_mapping.sql → 20_silver_views.sql
6. Data flows: Ignition → Connector → tag_events → silver_events_normalized → analytics
